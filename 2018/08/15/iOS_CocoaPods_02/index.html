CocoaPods 私有库

两种仓库

两种（ git ）仓库：spec repository 和 code repository。

  • spec repository

索引库（配置仓库）。

  1. 只用来存放 spec 文件，不存放代码。
  2. spec 文件是按照版本存放。每一个 spec 文件对应一个版本的代码。版本号就是代码库的 tag 号。
  3. 每一个 spec 文件包含对应版本代码库的信息（包括：仓库名、代码库的远程地址、版本号、依赖库、库文件等等）。
  4. spec 本地仓库路径：~/.cocoapods 。

每一个 spec 源都会在 ~/.cocoapods/repos/ 路径下对应一个本地 spec 仓库。其中，~/.cocoapods/repos/master 的 origin 是 https://github.com/CocoaPods/Specs.git ，就是 CocoaPods 官方的索引库。

  • code repository

代码仓库。

  1. 把开源库代码或 SDK 包文件上传到的仓库。
  2. CocoaPods 是通过索引文件（即spec文件）找到代码仓库的。

创建私有库

  • 1、在 git 远端创建索引仓库。

  • 2、在 git 远端创建代码仓库。所有代码仓库共用一个索引库。

  • 3、clone 代码库到本地，创建工程。一般包含：LICENSE（开源许可证）、README.md、name.podspec（CocoaPods 的索引描述文件）、代码（代码 & 二进制文件 & 资源文件）、Example（Demo工程）。

几个常用命令

执行命令时都做了什么事情：

  • 1、执行 pod setup 时，会从 github 上拉取 master 源。

  • 2、执行 pod install 时，会把 Podfile 中所有的私有库索引源对应的索引库拉取下来。

  • 3、执行 pod update 时，会更新本地所有的索引库。

  • 4、执行 pod repo udpate 时，更新本地所有索引库。
